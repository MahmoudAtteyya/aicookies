🚀 Quick Start

Fully OpenAI-compatible. Use the standard endpoint + your API key + any model from the list below.

POST https://aicookies.elliaa.com/v1/chat/completions

🔐 Authentication

All requests require a Bearer token:

Authorization: Bearer YOUR_AICOOKIES_API_KEY

cURL

curl -X POST https://aicookies.elliaa.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_AICOOKIES_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"mistral-small","messages":[{"role":"user","content":"Hello!"}]}'

Python (OpenAI SDK)

from openai import OpenAI
client = OpenAI(
    base_url="https://aicookies.elliaa.com/v1",
    api_key="YOUR_AICOOKIES_API_KEY"
)
response = client.chat.completions.create(
    model="mistral-small",
    messages=[{"role":"user","content":"Hello!"}])
print(response.choices[0].message.content)

Node.js

const res = await fetch("https://aicookies.elliaa.com/v1/chat/completions", {
  method:"POST",
  headers: {
    "Content-Type":"application/json",
    "Authorization":"Bearer YOUR_AICOOKIES_API_KEY"
  },
  body: JSON.stringify({model:"mistral-small", messages:[{role:"user",content:"Hello!"}]})
});

Streaming (SSE)

Add "stream": true to the request body for real-time token-by-token streaming:

curl -X POST https://aicookies.elliaa.com/v1/chat/completions \
  -H "Authorization: Bearer YOUR_AICOOKIES_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"mistral-small","stream":true,"messages":[{"role":"user","content":"Hello!"}]}'

🧠 Response Styles

⚡ Direct Response

# Request
curl ... -d '{"messages":[{"role":"user","content":"Say hello in Arabic"}]}'
# Response — just the answer
{"choices":[{"message":{"content":"مرحباً!"}}]}

All Mistral, Cohere, SambaNova, and kimi-k2p7-code are direct.

🧠 Reasoning Response

# Request — note higher max_tokens
curl ... -d '{"model":"glm-5p2","max_tokens":500,
  "messages":[{"role":"user","content":"Explain quantum computing"}]}'
# Response — thinking first, then answer
{"choices":[{"message":{"content":"1. Analyze: quantum computing uses qubits..."}}]}

Reasoning models: glm-5p2, qwen3p7-plus, deepseek-v4-pro

🔄 Smart Key Rotation

  1. Request → least-used active key → real provider API
  2. 429/402 → rotate to next key (free providers recover; Fireworks dies permanently)
  3. All keys busy → returns 503 Service Unavailable with retry_after_ms: 5000
  4. Response includes X-Proxy-Provider, X-Proxy-Key, X-Proxy-Latency headers
