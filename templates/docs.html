🧠 Response Styles Explained

⚡ Direct Response

Models that reply immediately — just the answer, no thinking process visible.

# Request
curl ... -d '{"messages":[{"role":"user","content":"Say hello in Arabic"}]}'

# Response — just the answer
{"choices":[{"message":{"content":"مرحباً!"}}]}

All Mistral, Cohere, SambaNova, and kimi-k2p7-code are direct.

🧠 Reasoning Response

Models that show their thinking process before answering. Important: set max_tokens high enough (≥200) or the answer may be cut off.

# Request — note higher max_tokens
curl ... -d '{"model":"glm-5p2","max_tokens":500,
  "messages":[{"role":"user","content":"Explain quantum computing"}]}'

# Response — thinking first, then answer
{"choices":[{"message":{"content":"1. Analyze: quantum computing uses qubits...
2. Key concepts: superposition, entanglement...
3. Answer: Quantum computing is..."}}]}

Reasoning models: glm-5p2, qwen3p7-plus, deepseek-v4-pro

🔄 Smart Key Rotation

  1. Request → least-used active key → real provider API
  2. 429 Rate Limit → rotate to next key (free providers recover)
  3. 402 Payment Required → Fireworks: 💀 DEAD permanently
  4. 401/403 Auth Error → disabled after 10 consecutive errors
  5. Response includes X-Proxy-Provider, X-Proxy-Key, X-Proxy-Latency headers
